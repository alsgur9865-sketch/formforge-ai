# FormForge AI — UI/UX 디자인 브리프

> **사용법**: 이 문서 전체를 [claude.ai/design](https://claude.ai/design) 에 붙여넣고
> "이 브리프로 FormForge AI 웹앱 화면들을 디자인해줘" 라고 요청하세요.
> 화면별 섹션(§5)을 하나씩 따로 요청하면 더 정교한 결과가 나옵니다.
> **본문은 한국어**, **UI에 실제로 들어갈 라벨/카피는 영어**(`英` 표기)로 적었습니다 —
> 글로벌 해커톤 심사 대상이라 UI chrome 은 영어 권장. (에이전트가 생성하는 토론 본문은
> 현재 한국어이며 데모에서 자막/병기 가능.)

---

## 1. 한 줄 소개

> **FormForge AI** — 두 AI 코치(격려파 vs 회의파)가 당신의 운동 영상을 보고 **실시간으로 토론**하고,
> Head Coach 가 합의를 내립니다. 당신의 피드백으로 코치들의 성격이 학습되어 **당신만의 코치**가 됩니다.

영어 태그라인(`英`): **"Two AI coaches debate your form. You referee. They evolve."**

---

## 2. 제품 개요

홈 피트니스 사용자가 운동 영상을 올리면:
1. **PoseExtractor** 가 자세를 정량 분석(관절 각도·반복 횟수·템포)
2. **The Encourager**(따뜻한 PT)와 **The Scrutinizer**(까다로운 운동생리학 박사)가 같은 영상을 **다른 관점**으로 보고 **여러 라운드 토론**
3. **The Mediator**(Head Coach)가 두 입장을 통합 + 과거 기록을 조회해 **최종 조언**
4. 사용자가 각 코치 톤에 피드백 → 다음 토론에서 코치 성격이 **진화**

기존 운동 자세 앱은 "단일 AI의 일회성 평가"인데, FormForge 는 **적대적 멀티 에이전트 토론 + 자기개선 루프 + 관측 가능성(observability)** 을 화면으로 보여주는 게 핵심 차별화입니다.

---

## 3. 이 UI가 특별해야 하는 이유 (디자인 목표)

해커톤 심사의 **Design** 점수를 잡는 화면이어야 합니다. 3가지를 시각적으로 "느껴지게":

1. **대결 구도(Adversarial)** — 두 코치가 좌우에서 맞붙는 라이브 토론. 라운드가 진행되며 서로를 반박하는 게 보여야 함. *정적인 채팅이 아니라 "토론"으로 읽혀야 한다.*
2. **합의(Consensus)** — 대결 후 Mediator 가 중앙에서 둘을 통합하는 "수렴" 모먼트.
3. **투명성/진화(Observability + Evolution)** — Phoenix trace(누가 언제 무엇을 호출했는지) 시각화 + 피드백으로 코치 성격 슬라이더가 움직이는 before/after.

> 한 문장: **"AI들이 내 앞에서 토론하고, 내가 그들을 길들이는 게 눈에 보이는 화면."**

---

## 4. 타겟 사용자 & 사용 맥락

- **누구**: 헬스장/홈트 사용자, 1인 트레이너 없이 폼 교정을 원하는 사람. 부상 이력이 있을 수 있음.
- **기기**: 데스크톱 웹(데모는 노트북 화면). 모바일 대응은 후순위(반응형이면 보너스).
- **세션 흐름**: 영상 업로드 → (분석 로딩) → 토론 관전 → 합의 확인 → 피드백 → (재방문 시) 진화 확인.
- **감정 곡선**: 호기심(업로드) → 몰입/긴장(토론 관전) → 안도/신뢰(합의) → 주도감(피드백으로 내가 코치를 바꿈).

## 5. 핵심 화면 (우선순위順)

### 5.1 ⭐ 메인 — 토론 라이브 뷰 (가장 중요)

**레이아웃**: 3-column 또는 좌/우 대결 + 하단 합의.

```
┌────────────────────────────────────────────────────────────┐
│  HEADER: FormForge AI 로고 · 운동타입 chip · 라운드 인디케이터  │
├──────────────────────┬─────────────────────────────────────┤
│  🟢 THE ENCOURAGER    │   🔴 THE SCRUTINIZER                │
│  (따뜻한 PT)          │   (운동생리학 PhD)                   │
│  ┌─ Round 1 ───────┐  │   ┌─ Round 1 ──────────────────┐    │
│  │ 칭찬 + 다음 스텝  │  │   │ primary_risk (severity 뱃지) │    │
│  └─────────────────┘  │   └────────────────────────────┘    │
│  ┌─ Round 2 ───────┐  │   ┌─ Round 2 ──────────────────┐    │
│  │ "회의파 의견에    │  │   │ "격려파가 놓친 점은..."      │    │
│  │  대해..." (반박)  │←→ │   (cross-reference 연결선)      │    │
│  └─────────────────┘  │   └────────────────────────────┘    │
├──────────────────────┴─────────────────────────────────────┤
│  ⚖️ CONVERGENCE: "두 코치가 '과도한 전방 기울기'에 합의"      │
├────────────────────────────────────────────────────────────┤
│  🧑‍⚖️ THE MEDIATOR (Head Coach) — 최종 합의 카드             │
└────────────────────────────────────────────────────────────┘
```

- **두 코치 칼럼이 시각적으로 대비**(색·아이콘·톤). Encourager=따뜻, Scrutinizer=차갑고 날카로움.
- 각 라운드는 **말풍선/카드 스택**으로 누적. Round 2+ 에서 상대를 언급하는 부분(`addresses_scrutinizer` / `addresses_encourager`)은 **연결선이나 인용 블록**으로 "반박 중"임을 표현.
- 상단 **라운드 인디케이터**: ●─●─○ (최대 3라운드). 합의되면 조기 종료.
- **로딩/진행 상태**: 분석·토론은 수십 초 걸림 → 단계별 진행("Analyzing pose… / Round 1… / Judging convergence… / Mediating…"). *스피너보다 "지금 무슨 단계"인지 보여주는 게 좋음.* (실시간 업데이트는 1초 폴링이라 약간의 지연 OK.)
- `英` 라벨: **"The Encourager" / "The Scrutinizer" / "The Mediator" / "Round 1/2/3" / "Consensus reached"**.

### 5.2 업로드 / 시작 화면

- 큰 **드래그&드롭 업로드 존**(`英` "Drop your workout video") + 운동 타입 선택(squat / deadlift / push-up) + (선택) 부상 이력 입력.
- 짧은 설명: 두 코치가 토론한다는 컨셉을 1-2줄 + 작은 일러스트/아이콘.
- 샘플 영상으로 바로 체험하는 **"Try a sample" 버튼**.

### 5.3 Mediator 합의 카드 (메인 하단 또는 별도 패널)

표시 데이터:
- **consensus**(2-3문장 통합 의견)
- **priority_actions**(순번 매긴 액션 1·2 + 각 rationale) — 체크리스트/스텝 형태
- **past_debate_references**(과거 토론 참조 — "지난번에도 같은 지적") → introspection 강조 뱃지
- **의료 면책 문구**(필수, §9)

### 5.4 Observability 패널 — Phoenix Trace 시각화

- 이 세션의 trace 를 **타임라인/트리**로: PoseExtractor → (Encourager ‖ Scrutinizer) → Convergence Judge → Mediator → MCP tool calls(`query_past_debates`, `query_similar_safety_flags`).
- 각 span 의 **latency·토큰·상태**를 작은 메트릭으로. "AI가 자기 과거를 조회했다"는 MCP 호출을 하이라이트.
- 톤: 개발자 대시보드 느낌(모노스페이스, 그리드). *심사위원에게 "진짜 멀티에이전트 + 관측성"임을 증명하는 화면.*
- `英`: **"Live trace" / "MCP introspection" / "p50 latency"**.

### 5.5 피드백 폼 (각 코치 톤 평가)

- Encourager: `너무 따뜻함 / 딱 좋음 / 너무 차가움` (`英` too warm / perfect / too cold)
- Scrutinizer: `너무 가혹함 / 딱 좋음 / 너무 밋밋함` (`英` too harsh / perfect / too soft)
- Mediator: 별점 1-5
- (선택) 자유 텍스트 한 줄.
- 세그먼트 버튼/슬라이더로 직관적이게. 제출 시 "코치가 학습 중…" 마이크로 인터랙션.

### 5.6 ⭐ Before / After — 코치 진화 (self-improvement 증명)

- 같은 운동을 피드백 전/후로 분석한 두 결과를 **나란히 비교**.
- **페르소나 파라미터 변화**를 슬라이더/게이지로: 예) Scrutinizer harshness `0.50 → 0.35` (애니메이션이면 강력).
- 톤 변화를 텍스트로도: "1주차(날카로움) vs 2주차(부드러워짐)".
- `英`: **"Week 1 vs Week 2" / "Your coach evolved"**.

## 6. 에이전트 페르소나 → 비주얼 언어

| 에이전트 | 성격 | 추천 비주얼 톤 |
|---|---|---|
| **The Encourager** | 따뜻한 PT(10년 경력), 동기부여 | 따뜻한 색(앰버/그린), 둥근 모서리, 부드러운 아이콘(예: 새싹·하이파이브) |
| **The Scrutinizer** | 까다로운 운동생리학 PhD, 직설·근거 기반 | 차가운 색(레드/슬레이트), 각진 카드, 데이터/경고 아이콘. severity 뱃지(low→critical 색 단계) |
| **The Mediator** | Head Coach, 균형·통합 | 중립(딥 블루/뉴트럴), 권위감, 합의 아이콘(저울·악수) |
| **PoseExtractor** | 백엔드 분석가(페르소나 없음) | 무채색·테크니컬(키포인트·각도 오버레이) |

> 두 코치의 **색 대비**가 이 앱의 시각적 정체성입니다. 토론이 "한 쪽 vs 다른 쪽"으로 즉시 읽혀야 함.

## 7. 실제 데이터 형태 (화면에 들어갈 실제 콘텐츠 — 분량 감 잡기용)

아래는 실제 시스템 출력 예시입니다. 컴포넌트가 담아야 할 텍스트 양/필드를 이걸로 가늠하세요.

**PoseExtractor 결과**:
```json
{ "exercise_type": "squat", "camera_angle": "side", "rep_count": 3,
  "overall_metrics": { "form_score_0_100": 65, "depth_consistency": 0.80 },
  "safety_flags": [ { "severity": "high", "issue": "excessive_forward_lean",
                      "rep_numbers": [1,2,3], "rationale": "요추 부담 — 부상 이력과 연결" } ] }
```

**Encourager(라운드 카드)**: praise(칭찬) + concern_one(딱 하나의 다음 스텝) + actionable_tip(구체 큐) + addresses_scrutinizer(반박, 2라운드+).

**Scrutinizer(라운드 카드)**: primary_risk{ name, severity(low/medium/high/critical), mechanism, evidence } + required_action + addresses_encourager.

**Mediator 합의(실제 예시)**:
- consensus: *"두 코치 모두 전방 기울기를 지적했습니다. 허리 부상 이력을 고려할 때 즉각 교정이 필요합니다…"*
- priority_actions: ① "중량 20-30% 감량" (rationale) ② "가슴을 펴고 상체 수직 유지" (rationale)
- past_debate_references: [{ debate_id, date, outcome: "과도한 전방 기울기로 인한 요추 부상 위험" }]
- disclaimer: *"정보 제공용입니다. 의학 조언이 아닙니다…"*

> 카드 1개당 텍스트가 **2-4문장 + 뱃지 1-2개** 수준. 너무 빽빽하지 않게, 스캔 가능하게.

## 8. 브랜드 & 무드 (추천 + 대안)

**제품명 의미**: *Forge(대장간)* — 자세를 "단련/벼린다". 망치·불꽃·금속 메타포 약간.

**추천 무드 (A안 — Dark Tech Arena)**:
- 다크 모드 베이스(딥 차콜/네이비), 두 코치 색이 강하게 빛나는 대비.
- 모던·테크·약간의 스포츠 에너지. 토론이 "아레나/링"처럼 느껴지게.
- 타이포: 굵은 산세리프 헤드라인 + 가독성 좋은 본문. 데이터는 모노스페이스.
- 모션: 라운드 등장 시 슬라이드, 합의 시 두 칼럼이 중앙으로 수렴하는 트랜지션.

**대안 (B안 — Clean Clinical Trust)**:
- 라이트 모드, 화이트/소프트 그레이 + 코치 액센트 색. 의료·신뢰감.
- 차분하고 깔끔. 부상 예방·전문성 강조.

> 기본은 **A안(Dark Tech Arena)** 추천 — 토론/대결 컨셉과 가장 잘 맞고 데모 영상에서 임팩트가 큼.
> 컬러는 취향대로 조정하되 **두 코치의 명확한 색 대비**만 지켜주세요.

## 9. 필수 요소 (절대 빠지면 안 됨)

- **의료 면책 문구**: 모든 분석 결과 화면에 *"정보 제공용. 의학 조언 아님. 통증·부상 시 전문가 상담."* (`英` "For information only — not medical advice.") 항상 보이게(푸터 또는 결과 카드 하단).
- **두 코치는 항상 분리/대비**되어 보여야 함(단일 응답처럼 보이면 안 됨 — 컨셉 훼손).
- **MCP introspection 가시화**: Mediator 가 "과거를 조회했다"는 게 드러나야 함(past_debate_references / trace 패널).
- **로딩 중에도 무슨 단계인지** 표시(수십 초 소요).

## 10. 기술 제약 (디자인이 알아야 할 것)

- **구현은 Streamlit**(Python). 따라서:
  - 컴포넌트는 Streamlit 으로 만들 수 있는 범위(컬럼, 컨테이너, 카드, 메트릭, 차트, 프로그레스, 세그먼트 버튼, expander)를 선호.
  - 업데이트는 **1초 폴링 방식**(실시간 push 아님) → "라이브"하되 1초 단위 갱신 정도의 부드러움이면 충분.
  - 아주 복잡한 커스텀 캔버스 애니메이션보다, **명확한 레이아웃 + 색/타이포/뱃지 + 가벼운 CSS 트랜지션**이 현실적.
- 디자인 결과물은 **목업/레퍼런스**로 쓰고, 실제 구현은 Streamlit + custom CSS 로 옮길 예정. → "Streamlit 으로 재현 가능한" 디자인이면 가장 좋음.
